Data Structures

Not only for contest programming, data structures in computer science are very significant. It is very important to know the properties of each data structure to know which one should be implemented. Let's start with some of the basic data structures:


Array

This is the most primitive data structure. It is a collection of elements of the same type (int, char, string etc.), and each element can be identified by a position number/index. It is important to remember that indexing in computer science is always 0-based! (Index starts from 0). Arrays can be single or multidimensional.

Examples:
C++: int arr[500];
Java: int[] arr = new arr[500];
Python: arr = [0] * 500
(Of course, the data types can be modified to whatever you want)


Dynamic Array

A dynamic array is like an ordinary array except that the size of it can be changed constantly. That is why it is dynamic, because it can change. This is useful if the size is not known, or if memory wants to be used as efficient as possible.

Examples:
C++[1]: vector<int> dyn_arr;
Java[2]: Arraylist<Integer> dyn_arr = new Arraylist<Integer>();


Queues

A queue is what it sounds like, a "line". In a line, the first person to go to it is the first to come out. The queue behaves the same way. Also known as the FIFO (First In, First Out), the first elements added / pushed into the queue are the ones to come out first. This is useful if the order that data is entered needs to be preserved.

Examples:
C++[3]: queue<int> q;
Java[4]: Queue<Integer> q = new LinkedList<Integer>();


Stacks

Stacks are the opposite of queues, and a stack is also known as a LIFO (Last In, First Out). The last element that is added / pushed will be on the top of the stack, so that it will be the first to be removed. This can be useful in many different scenarios.

Examples:
C++[5]: stack<int> stk;
Java[6]: Stack<Integer> stk = new Stack<Integer>();


Sets

Sets contain is a collection of unique elements, so that if duplicates are entered, only one will exist in the set. (The others will be discarded). The elements in a set are naturally sorted, which can be very useful.

Examples:
C++[7]: set<int> s;
Java[8]: Set<Integer> s = new HashSet<Integer>();


Hash Tables / Hash Maps / Maps

These are a one-to-one data structure, that maps/links one element to another. This can be useful for storing information between two different data types.

Examples:
C++[9]: map<string, int> mp;
Java[10]: HashMap<String, Integer> mp = new HashMap<>();


Pairs (C++ Only)

Pairs are a useful data structure that is only available in C++. However, you can create your own "pair" data structure in Java. The C++ pair has two elements. Let's say you have a pair called foo. To access the first element of foo, use foo.first. To access the second, use foo.second.

Examples:
C++[11]: pair<int, int> foo;



These are the basic data structures to learn, and it is a great idea to fully understand how to use each one. This can be done by practicing a lot of problems (any type, since nearly all problems utilize a data structure). You can even make your own data structure! Good luck and keep practicing.
